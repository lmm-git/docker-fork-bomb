# docker-fork-bomb

Strange things happen, when you execute `docker run -it --rm --memory=128M --kernel-memory=16M lmmdock/fork-bomb`. Be aware that your complete system may crash, even your file system might get corrupted by this command.

It basically just executes a fork bomb in a resource (memory) constrained container.

## Syslog of exemplary run

Executing the image with `--oom-score-adj=900` allowed us to test the behavior without affecting other services. Doing so, the following output was generated by `dmesg -Tw`.

_Note_: Without adjusting the OOM score, other services (such as other containers, user applications on the host or even the file system drive of the host) were affected.
